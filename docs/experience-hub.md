# Neo N4 Experience Hub

Open the local-first Experience Hub:

[Launch the Neo N4 Experience Hub](./experience-hub/index.html)

The hub is a read-only product surface for understanding, building, operating,
and verifying Neo N4. It visualizes NeoHub deployed L1 contracts, NeoFS DA,
ContractZkVerifier, the deployable L1 ZK verifier contract, Gateway, SharedBridge, L2
NeoVM2/RISC-V execution, optional N4 L2 VM profiles, and validation evidence.

The browser does not hold private keys or sign deployment/governance actions.
Privileged actions remain in CLIs, wallets, node processes, and contracts.

For the mathematical foundation behind the execution and proof path, open the
companion [Neo N4 Math Lab](./interactive-math/index.html). It teaches finite
fields, commitments, NeoVM stack semantics, NeoVM2/RISC-V cycles, zkVM
arithmetization, proof verification, aggregation, and N4 security checks.

The first phase uses deterministic local reports and a generated repository
manifest. Real public testnet evidence must come from later deployment
rehearsals and CI artifacts; private devnet evidence is labeled as such in the
UI.
