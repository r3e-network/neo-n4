# L2 支付中继

外部桥的 L2 支付路径是异步流程。`ExternalBridgeEscrow` 调用不可变的
`NeoHub.L2PayoutAdapter`；适配器只有在认证托管合约、逐字段校验委员会签名的规范字节、
并持久化队列记录后才返回 `true`。这不代表 L1 交易与目标 L2 入账具有跨链原子性。

## 绑定的指令

中继原样传递 `ExternalMessageHasher.EncodeCanonical` 字节。其 `Hash256` 依次绑定来源外部链、
目标 Neo 链、nonce、方向、外部发送者、收款人、截止时间、来源交易、消息类型和负载。
支付负载还绑定外部资产及最小编码的金额；托管路由在签名字节之外绑定映射后的 Neo 资产。
适配器、中继解码器和 L2 原生入口都会独立拒绝字段或哈希不一致；当前只支持纯资产转账。

## 持久状态

| 状态 | 已持久化事实 | 下一项副作用 |
| --- | --- | --- |
| `Enqueued` | 已确认的适配器事件及完整指令写入 RocksDB | 构建目标 L2 交易 |
| `CreditPrepared` | 已签名的目标 L2 交易先写日志 | 广播或对账 |
| `Credited` | 原生收据保存消息哈希和 L2 交易哈希 | 构建 L1 确认交易 |
| `AcknowledgementPrepared` | 已签名的 L1 确认交易先写日志 | 广播或对账 |
| `Acknowledged` | 适配器保存准确的 L2 交易哈希 | 终态，无副作用 |
| `Poisoned` | 有界重试耗尽并保留最后错误 | 运营方修复 |

每次重启都会先读取链上状态，再决定是否重播。原生入口认证固定中继账户，校验目标链和本地
资产映射，在通过 `BridgedNep17` 铸币前保存收据，并且只消费一次
`(externalChainId, nonce)`。适配器确认同样要求固定中继见证，并对准确哈希保持幂等。

## 信任假设

- L1 最终性以及两个 RPC 端点必须返回规范链状态。
- 固定中继签名者控制提交和可用性，但无法通过该入口篡改字段或铸造未注册映射。
- L2 验证者集合及 r3e Neo 核心分支必须执行所配置的原生合约代码。
- 中继停止只会延迟入账和确认，不会把排队伪装成已确认；重放也不会产生第二次余额增加。
