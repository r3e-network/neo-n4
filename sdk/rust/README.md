### Technical Role

- **Layer:** Developer SDK
- **Purpose:** Rust client SDK for building tools and services that talk to Neo N4 APIs.
- **Inputs:** developer app | gateway endpoint | wallet/config
- **Responsibilities:** Encode API requests | Handle bridge/proof models | Return typed results
- **Outputs:** typed client result | transaction request | query response
- **Consumers:** apps | operators | integration tests

### Reading Order

1. Start with system position and conceptual architecture.
2. Read technical principles, trust boundaries, and state model to understand correctness.
3. Follow workflow and dataflow to see runtime movement.
4. Use proof/evidence flow, integration map, and lifecycle for operational understanding.
